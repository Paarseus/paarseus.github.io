Autonomous Navigation & Path Planning

Parsa Ghasemi1, Behnam Bahr1
1Cal Poly Pomona
Part of DRIVE Platform
AVROS

ROS2 Nav2 integration with Dynamic Window Approach, Pure Pursuit path following, and OpenStreetMap-based GPS waypoint navigation.

Abstract

We present the autonomous navigation and path planning system for a full-scale autonomous vehicle built on ROS2 Nav2. The system integrates OpenStreetMap routing via OSMnx with Dijkstra's shortest path algorithm for GPS-based waypoint navigation. Local planning uses a Dynamic Window Approach adapted for Ackermann-steering vehicles with a kinematic bicycle model and multi-objective trajectory optimization. Path following is handled by a Pure Pursuit controller with adaptive speed-dependent lookahead distance. The complete navigation stack comprises 1,600+ lines of code with 8 configurable launch files.

ROS2 Nav2 Integration

The navigation system is built on ROS2 Nav2 with SmacPlannerHybrid for kinematically feasible global path planning. Custom lifecycle management ensures clean startup and shutdown of all navigation components. A modular launch architecture with 8 configurable launch files enables selective activation of sensor, localization, and navigation subsystems. The system interfaces with robot_localization for EKF-based pose estimation and tf2 for coordinate frame management.

Dynamic Window Approach

The local planner implements a Dynamic Window Approach adapted for Ackermann-steering vehicles. A kinematic bicycle model constrains the trajectory search space to physically feasible motions. Multi-objective trajectory optimization balances goal heading, path clearance, and velocity to select the best trajectory at each planning cycle. The planner accounts for the vehicle's minimum turning radius and maximum steering rate.

Pure Pursuit Controller

Path following uses a Pure Pursuit controller with a geometric steering law that computes the required steering angle to reach a lookahead point on the planned path. The lookahead distance adapts based on vehicle speed—longer at higher speeds for stability, shorter at low speeds for precision. The controller interfaces with the distributed CAN bus control system to execute steering commands at 100 Hz.

GPS-Based Routing

The route planner queries OpenStreetMap data via OSMnx to compute Dijkstra shortest paths between GPS coordinates. Waypoints are extracted from the computed route and fed to the Nav2 NavigateThroughPoses action server. This enables the vehicle to navigate real road networks using GPS positioning, bridging the gap between global GPS-level routing and local sensor-based navigation.
