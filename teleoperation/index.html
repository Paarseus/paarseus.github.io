Remote Vehicle Teleoperation System

Parsa Ghasemi1
1Cal Poly Pomona
Part of DRIVE Platform
AVCONSOLE

Drive the vehicle from a simchair, a phone, or your voice—WebSocket streams commands while WebRTC pipes back live video.

Abstract

We present a multi-modal remote teleoperation system for a full-scale autonomous vehicle. The system supports three input modalities: a physical simchair with steering wheel and pedals, a mobile phone WebUI, and voice commands via on-device Whisper speech-to-text. A WebSocket bridge streams control commands to the CAN master controller in real-time, while WebRTC provides low-latency live video from multiple vehicle cameras. A priority arbitration system safely manages concurrent inputs from multiple sources, with the simchair taking highest priority and voice commands the lowest.

WebSocket Architecture

The teleoperation system uses a WebSocket bridge to stream control commands from remote operator interfaces to the vehicle's CAN master controller. Commands are serialized as JSON messages and transmitted over a Tailscale VPN connection, providing secure remote access without exposing the vehicle's network. The WebSocket server on the vehicle receives commands and translates them into CAN bus messages at the master controller's 100 Hz update rate.

Video Streaming

WebRTC provides low-latency live video streaming from multiple vehicle cameras to the operator interface. The system supports configurable camera selection and resolution, with adaptive bitrate based on network conditions. The operator WebUI displays camera feeds alongside vehicle telemetry data including speed, steering angle, and system health indicators.

Voice Control

Voice control runs OpenAI's Whisper model on-device for speech-to-text conversion, eliminating cloud dependency and latency. Recognized commands are parsed into vehicle control actions (forward, stop, turn left/right, speed up/slow down). The voice pipeline includes confirmation feedback and safety constraints that limit maximum speed and require explicit confirmation for potentially dangerous maneuvers.

Priority Arbitration

When multiple input sources compete for vehicle control, a priority arbitration system ensures safe behavior. The simchair has highest priority as the primary operator interface, followed by the phone WebUI, with voice commands at the lowest priority. Input transitions include timeout-based fallback—if the active input source stops sending commands, control gracefully transfers to the next available source or triggers an emergency stop.
