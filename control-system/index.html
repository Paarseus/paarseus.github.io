Distributed Embedded Vehicle Control System

Parsa Ghasemi1, Aspen Knox1
1Cal Poly Pomona
SCCUR 2025 · Oral Presentation
Control Code PCB Design CAN Suite

Four Teensy 4.1 MCUs on a CAN bus running closed-loop steering, throttle, and brake control with zero safety incidents over 50+ hours.

Abstract

We present a distributed embedded control system for a full-scale autonomous vehicle. Four Teensy 4.1 microcontrollers communicate over a CAN bus at 250 kbps to run closed-loop steering, throttle, and brake control. Written in 4,900+ lines of C++, the system implements a Master CAN Controller, MCP4728 quad-DAC throttle output over I2C, stepper motor steering with limit switch protection, and linear actuator brake control with position feedback.

A four-layer E-STOP architecture and 500ms hardware watchdog have maintained zero safety incidents through 50+ hours of autonomous driving. The entire system runs on a custom PCB designed in KiCad with SN65HVD230 CAN transceivers, TXB0104D level shifters, and Traco DC-DC converters.

CAN Bus Architecture

The distributed system coordinates 4 specialized microcontroller nodes. A Master CAN Controller runs at 100 Hz and arbitrates commands from the autonomy stack, teleoperation interfaces, and the E-STOP system. A throttle node drives an MCP4728 quad DAC over I2C for precise voltage output (0.3V–4.0V) with drive mode selection. A steering node controls a stepper motor with analog encoder feedback and non-blocking pulse generation. A brake node implements closed-loop linear actuator control with stroke calibration. Custom CAN message protocol (0x100/0x200/0x300) provides JSON telemetry and ASCII command interface for remote monitoring.

Safety System

The four-layer E-STOP architecture provides defense in depth: a physical kill switch, CAN-level software E-STOP, watchdog timeout failsafe, and per-actuator safety limits. If any node loses CAN communication for more than 500ms, the hardware watchdog triggers automatic emergency braking across the entire network. This architecture has achieved <200ms emergency response time and zero safety incidents over 50+ hours of autonomous operation.

Custom PCB Design

The custom PCB designed in KiCad integrates SN65HVD230 CAN transceivers for reliable differential signaling, TXB0104D level shifters for mixed-voltage logic interfacing, and Traco DC-DC converters for power regulation. The board provides clean power distribution, sensor connector breakouts, and a compact form factor for mounting in the vehicle.

CAN Bus Cybersecurity

A multi-layered cybersecurity framework hardens the CAN bus network against injection attacks, replay attacks, and unauthorized access. The defense-in-depth security architecture layers message authentication, anomaly detection, and fail-safe isolation onto the CAN bus. This work was presented as an Oral Presentation at the Southern California Conference for Undergraduate Research (SCCUR 2025).
